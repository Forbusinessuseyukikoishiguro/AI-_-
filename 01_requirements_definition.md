# うさうさ読書手帖 要件定義書（v8.0）

🐰 うさうさ研修工房 ｜ 単一HTML・オフライン読書記録ツール

---

## 1. 背景・目的

読書週間の習慣化を支援する、軽量な個人向け読書記録ツール。インストールやアカウント登録なしで、すぐ使い始められることを最優先とする。学習メモは「定義→たとえ→詳細→一言まとめ」の四段階で残し、自分の言葉で理解を定着させる。

**設計の根っこ**：1ファイル・オフライン・外部依存なし。配布・改修・バックアップが「ファイル1個」で完結することを価値の中心に置く。

---

## 2. スコープ

### 対象
- 1ユーザー・1ブラウザ内で完結する読書記録／習慣化支援
- ローカル（`localStorage`）でのデータ永続化
- 書きかけ入力の保護、エラーの可視化、週次の自動リセット

### 対象外
- サーバ同期・複数端末間共有・アカウント認証
- 複数ユーザーの同時利用
- クラウドバックアップ（手段はMarkdown手動出力のみ）

---

## 3. 用語

| 用語 | 定義 |
|------|------|
| 手帖（ノート） | 四段階で綴じた1枚の学習メモ |
| 桜／つぼみ | 週7日の読書達成マーカー（咲く🌸＝達成、つぼみ🪷＝未達成） |
| 閉じた状態（weekBare） | リセット後、つぼみを消した空きマス状態 |
| 書きかけ（下書き） | 未確定の入力欄テキスト。確定データ（手帖）とは区別する |
| 和菓子 | 達成のごほうび。称号・継続のゲーミフィケーション要素 |

## 5. 非機能要件（NFR）

| ID | 区分 | 要件 |
|----|------|------|
| NFR-01 | 配布性 | 単一HTMLファイルで動作。ビルド・サーバ・CDN依存なし |
| NFR-02 | オフライン | ネットワーク非接続で全機能が動作 |
| NFR-03 | 技術制約 | vanilla JS のみ。フレームワーク・外部ライブラリを使わない |
| NFR-04 | データ保全 | `localStorage` 失敗時はメモリへ退避し、入力を即失わない |
| NFR-05 | 可観測性 | 例外・保存失敗を時刻つきで記録し、タブを閉じても残す |
| NFR-06 | 復旧性 | 書きかけはタブ／ブラウザを閉じても復元できる |
| NFR-07 | 時刻仕様 | 自動リセットはローカル時刻の日曜23:59:59（JST想定・DSTなし前提） |
| NFR-08 | 互換性 | モダンブラウザ（ES5相当の書き方で広く動作）。`localStorage` 無効環境でも致命停止しない |
| NFR-09 | 保守性 | 時刻計算は単一関数に集約し、表示・タイマーが同じ真実を参照 |
| NFR-10 | 検証性 | 実ファイルを jsdom で読み込み自動テスト可能であること |

## 6. データ要件

`localStorage` キー一覧：

| キー | 用途 |
|------|------|
| `usausa_techo_v1` | 確定状態（state本体） |
| `usausa_techo_input_v1` | 書きかけ入力（下書き） |
| `usausa_techo_errlog_v1` | エラーログ（最大50件） |
| `usausa_techo_draft_v1` | 手動一時保存（state） |
| `usausa_techo_auto_v1` | 自動一時保存（state） |
| `usausa_autoerase_v1` | 手帖自動消去ON/OFF |

state スキーマ：

```
{ wagashi:number, notes:Note[], morningDate:string|null, readDate:string|null,
  weekStart:string, weekDone:boolean[7], weekBare:boolean, streak:number, lastDay:string|null }
Note = { id:number, title, def, meta, detail, sum, date }
```

---

## 7. 制約・前提

- ローカル時刻は利用者ブラウザ依存。本仕様はJST（DSTなし）を前提に設計。
- データは端末ローカルのみ。端末/ブラウザのストレージ削除で消える（バックアップはMarkdown手動出力）。
- `setTimeout` の1週間遅延は上限（約24.8日）内のため単発タイマーで成立。
