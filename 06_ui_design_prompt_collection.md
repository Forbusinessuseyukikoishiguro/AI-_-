## A. ビジュアルデザインの土台

### A-1. デザイントークン（配色・テーマをCSS変数に集約）
**要点**：色・影・線を `:root` のCSS変数に集約。テーマ変更も一括。
**使いどころ**：世界観を持たせたい全ツール。

```
配色・影・罫線をすべて :root のCSS変数に集約して。
- 主要色/補助色/背景/墨(文字)/線/影 を変数化（例 --paper, --ink, --line, --shadow）
- 以降の各パーツはこの変数だけを参照（ハードコード色を残さない）
- 後でテーマを足せるよう、色の意味で命名（用途名。具体色名にしない）
```

### A-2. 世界観のある配色とフォント
**要点**：テーマに沿った限定パレット＋見出しに専用フォント（明朝など）で空気感を出す。
**使いどころ**：無印UIから脱したいとき。

```
{テーマ：例 和/レトロ/夜カフェ}の世界観で配色とフォントを決めて。
- パレットは5〜7色に絞る（多用しない）
- 見出しは{雰囲気フォント：例 明朝}、本文は可読性重視のサンセリフ
- 彩度・明度をテーマで統一し、装飾は控えめに上品に
配色の意図を一言ずつ添えて。
```

### A-3. 紙・質感のある背景
**要点**：単色ベタ塗りでなく、放射グラデ＋淡いテクスチャで“素材感”を出す。
**使いどころ**：のっぺり感を消したい背景。

```
背景に質感を持たせて。
- radial-gradientで中心から外へ淡く変化させる
- ベタ塗りを避け、紙/和紙のような生成りトーン
- 重くしない（画像は使わずCSSグラデのみ）
```

## B. UIコンポーネント

### B-1. カード型レイアウト
```
各セクションをカード化して。角丸・淡い境界線・やわらかい影、見出しに小さな印(シール)アイコン。
余白を広めにとり、情報のまとまりごとに分割して。
```

### B-2. トースト通知
**要点**：画面下中央にスッと出てフェードで消える短い通知。多用しない。
```
操作結果を伝えるトーストを入れて。
- 画面下中央、translateY＋opacityでスライドイン、数秒で自動消滅
- 連続呼び出しはタイマーをクリアして上書き
- pointer-events:none で操作を邪魔しない。{絵文字}を添えて短く
```

### B-3. カスタムトグルスイッチ
```
チェックボックスをカスタムのトグルスイッチに。
- 実体は input[type=checkbox] を隠し、label+spanのスライダーで表現
- checked時に色とつまみ位置をtransitionでアニメ
ラベルテキストとアクセシブルな関連付け(for/id)も付けて。
```

### B-4. 吹き出し（CSS擬似要素のしっぽ）
**要点**：`::after` の三角形でしっぽを作り、`drop-shadow`で枠線も繋げる。
```
キャラのセリフ用に吹き出しを作って。
- ::after の border三角形でしっぽ、filter:drop-shadowで枠線を連続させる
- 角丸・淡い枠・やわ影。中の文字は一部を強調色にできるように
```

### B-5. 落款・シールなどの装飾
```
要所に和の装飾を足して（重くしない範囲で）。
- 見出し横の小さな朱印(角丸の塗り＋白文字一字)
- フッターに枠線囲みの落款風マーク
装飾はコンテンツの可読性を邪魔しないこと。
```

### B-6. 空状態（empty state）
**要点**：データ0件のときに、淡い案内文で次の一歩を促す。
```
データが0件のとき専用の空状態を表示して。
- 「まだ{対象}は空です。最初の一つを{動詞}してみましょう」程度の淡い案内
- 関連ツール(全選択/一括削除等)は0件時に隠す or 無効化
```

## C. マイクロインタラクション / 動き（つぼみの動き等）

### C-1. 状態で見た目が変わる要素（つぼみ→咲く→閉じる）
**要点**：1要素に複数状態（未達/達成/閉）を持たせ、状態クラスで描画とアイコンを切替。
**使いどころ**：達成マーカー・進捗ドット・チェックリスト。

```
{達成マーカー}を「状態で見た目が変わる」要素にして。
- 状態：未達成(つぼみ🪷) / 達成(咲く🌸) / 閉じた(空きマス) の3つ
- 状態はクラスで分岐し、アイコンと背景を切替（done / closed など）
- クリックで未達成⇄達成をトグル、達成時にごほうび処理を呼ぶ
- リセットは「閉じた状態」にして、次の周期で通常表示に戻す
```

### C-2. クリック時のpop / hoverの拡大
**要点**：`@keyframes`の弾むスケール（0.2→1.2→1）と、hoverの軽い拡大で“手応え”を出す。
```
押した時に弾むpopアニメ、hoverで軽く拡大する手応えを足して。
- pop: scale 0.2→1.2→1 のkeyframes、達成の瞬間に付与
- hover: transform: scale(1.06) を短いtransitionで
やりすぎない（0.15〜0.4s程度）。
```

### C-3. 環境アニメ（背景に舞う花びら）
**要点**：装飾要素をJSで複数生成し、`animation-delay`/`duration`/位置をランダム化。`pointer-events:none`で操作を邪魔しない。
**使いどころ**：常時の“生きている感”。

```
背景に{舞うもの：花びら/雪/木の葉}をアンビエント表示して。
- JSで十数個を生成し、left/サイズ/animation-duration/delayをランダム化
- 上から落ちて回転しながらフェードする keyframes（loop）
- position:fixed・pointer-events:none・低z-index で操作を妨げない
- 数は控えめにして軽さを保つ
```

### C-4. ふわふわ浮遊 / 灯りのゆらぎ
```
{キャラ/灯り}に常時の微アニメを。
- 浮遊: translateYを±数pxで往復(ease-in-out, 5s程度)
- ゆらぎ: box-shadowの強さを周期変化させて灯火っぽく
無限ループでもCPUに優しい範囲で。
```

### C-5. SVGキャラの“もこもこ”質感
**要点**：`feTurbulence`＋`feDisplacementMap`で輪郭をゆらし、ふわっとした質感を出す。
**使いどころ**：手描き風・ぬいぐるみ風のSVG。

```
SVGキャラにもこもこ質感を付けて。
- feTurbulence(fractalNoise)＋feDisplacementMapのfilterで輪郭を微妙にゆらす
- 体/頭/耳を重ねた楕円で構成し、淡いハイライト楕円を重ねる
- 目はまん丸＋小ハイライト、ほっぺに淡いピンク
重くなりすぎないようscaleは控えめに。
```

## D. テーマ切り替え

### D-1. bodyクラスでモード切替（＋演出）
**要点**：`body.cafe` のようにクラス1つで配色を切替、モード限定の演出（湯気など）も足す。
```
表示モード切替を入れて（例：通常 ⇄ {モード名}）。
- トグルで body にクラスを付け外し、配色をそのクラス配下で上書き
- そのモード限定の小演出（例：立ちのぼる湯気アニメ）を追加
- 切替状態は保存して次回も復元
```

## E. 自動時間削除（TTL / 期限切れ）

### E-1. 作成からN日で自動消去（起動時sweep）
**要点**：各レコードに作成時刻を持たせ、起動時に「now - 作成 > TTL」を除外。除外件数を通知。
**使いどころ**：一時メモ・ログ・キャッシュの自動掃除。

```
{対象レコード}の自動消去を入れて。
- 各レコードに作成時刻(タイムスタンプ)を保持
- 起動時に now - 作成 > {N}日 のものを除外（sweep）、除外件数を返してトースト通知
- 作成時刻が無い旧データは安全側で残す
- TTLは定数で一箇所に（NOTE_TTL_DAYS のように）
```

### E-2. ON/OFFトグル＋確認＋遡及処理
**要点**：自動消去はデータ破壊なので、ONにする瞬間にconfirmし、既存の期限切れも即時sweep。
```
自動消去のON/OFFスイッチを付けて。
- ONにする時はconfirmで「{N}日経過分が自動で外れる」旨を確認
- ON直後に既存の期限切れを即sweepして件数を通知
- 設定は保存。OFFでも既に消えたものは戻らない旨を明示
```

### E-3. 期限ベースの自動リセット（時刻系へ）
**要点**：「毎週日曜23:59にリセット」のような期限処理は時刻スケジューラ側で。詳細は前回集の C 章（純粋関数集約＋取りこぼし救済）を参照。
```
{周期}での自動リセットは、次回時刻を純粋関数1個に集約し、
visibilitychange/focusで取りこぼしを救済する方式で。（時刻系プロンプト集C章を流用）
```

## F. ゲーミフィケーションのフィードバック

### F-1. 達成で+1（pop付き）
```
達成のたびに{ごほうび：和菓子/スター}が1つ増える棚を。
- グリッドに並べ、増えた要素にpopアニメ
- 上限到達やマイルストンでトースト通知
```

### F-2. 称号・継続のチップ表示と昇格通知
**要点**：累計に応じた称号テーブルと連続日数を小チップで常時表示、昇格時に通知。
```
称号と継続日数のチップをヘッダーに常時表示して。
- 称号は[必要数, 名称]のテーブルで判定、昇格した瞬間にトースト
- 継続は「前回が昨日なら+1、今日なら据置、それ以外は1にリセット」
```

## G. アクセシビリティの小物
```
装飾と意味を分けて最低限のa11yを入れて。
- 純装飾のSVG/演出は aria-hidden、意味のある画像は role=img + aria-label
- 動的に変わる通知領域は aria-live="polite"
- トグル/ボタンはラベルと関連付け
```

## H. まとめて立ち上げ（世界観つきUIブートストラップ）

```
{ツール名}のUIを{テーマ}の世界観で組んで。最初から以下を入れて：
1. 配色・影・線を:root変数に集約（用途名で命名）
2. カード型レイアウト＋トースト＋カスタムトグル＋空状態
3. 達成マーカーは「未達/達成/閉じた」の3状態、クリックでpop、hoverで微拡大
4. 背景に控えめなアンビエントアニメ（pointer-events:none）
5. bodyクラスでテーマ切替＋モード限定演出
6. 装飾はaria-hidden、通知領域はaria-live
重くしない・やりすぎない・色はハードコードしないこと。
```
