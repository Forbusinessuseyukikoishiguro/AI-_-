## この記事の使い方

`usausa_dokusho_techo_v8.html`（単一HTML・オフライン・vanilla JS）を、**AIへのプロンプトだけで再現**するための手引きです。理解しながら作れるよう、各ステップに【解説】を付けています。使い方は2通り。

- **モードA：一括再現** … 「共通ルール」＋「マスタープロンプト」を1回貼る。最速。
- **モードB：段階再現** … 12フェーズを順に貼る。各フェーズの【解説】で設計意図を理解しながら積み上げる。学習・改造向き。

> どちらでも、最後に「商用テスト→バージョンアップ」のプロンプトで仕上げます。

## 最初に貼る：共通ルール

```
このプロジェクトの共通ルール（以後ずっと前提）：
- 出力は単一HTMLファイル（HTML/CSS/JSを1ファイルに同居）
- オフラインで全機能が動く。サーバ・ビルド・外部CDN・フレームワーク禁止。vanilla JSのみ
- データはlocalStorage。状態はDOM操作で描画
- 色・影・線はすべて:root のCSS変数に集約（用途名で命名、ハードコード色禁止）
- 破壊的操作（全消去・一括削除・自動消去ON）はconfirmで確認
- 例外・保存失敗は握りつぶさず、後述のエラーログに記録
- 嘘をつかない。曖昧な箇所は推測で埋めず、解釈を明示してから実装
世界観は「和（わ）」。生成りの和紙、明朝見出し、朱・金茶・抹茶・桜の限定パレット。
```

【解説】最初に制約と世界観を固定すると、以後の指示が短く済み、生成のブレが減ります。「単一HTML／vanilla／オフライン」は今回の根幹なので必ず宣言します。

# モードB：段階再現（12フェーズ・解説つき）

## フェーズ1：土台とデザインシステム

【解説】最初に「色の地図（デザイントークン）」と世界観を敷きます。ここを変数化しておくと、後のパーツが全部この変数を参照でき、テーマ変更も一括になります。

```
共通ルール前提で、土台のHTML/CSSだけ作って。
- :root に和のパレットを変数で定義：--paper(生成り) --washi(和紙) --ink(墨) --line --shadow
  --sakura --matcha --vermilion(朱) --gold(金茶)、明朝用クラス .mincho
- bodyは radial-gradient の和紙風背景。最大760px中央、縦1カラム、カード型の枠を用意
- 角丸・淡い境界・やわらかい影の .card を定義
まだ機能は入れず、空のセクション枠だけ。
```

## フェーズ2：ヘッダー＋うさうさSVG＋吹き出し

【解説】世界観の“顔”を作ります。うさぎは `feTurbulence`＋`feDisplacementMap` で輪郭をゆらして“もこもこ”質感に。吹き出しのしっぽは `::after` の三角＋ `drop-shadow` で枠線を繋ぎます。

```
ヘッダーと応援エリアを足して。
- ヘッダー：行灯風の灯り(box-shadowでflickerアニメ)＋タイトル(明朝)＋称号/継続チップ
- うさうさ：SVGのもこもこ白うさぎ。feTurbulence+feDisplacementMapのfilterで輪郭をゆらす。
  体/頭/耳を楕円で重ね、まん丸の目＋ハイライト、淡いほっぺ、本を持つポーズ。常時ふわふわ浮遊(bob)
- 吹き出し：::afterの三角でしっぽ、drop-shadowで枠線を連続。中の応援文と「べつのことば」再抽選ボタン
- 応援文は十数パターンから抽選（穏やかな励まし。説教臭くしない）
```

## フェーズ3：読書週間の桜（つぼみの動き・3状態）

【解説】今回の象徴。1マスに「未達(つぼみ🪷)／達成(咲く🌸)／閉じた(空きマス)」の3状態を持たせ、**状態クラスで描画を分岐**します。データは `weekDone:boolean[7]` ＋「閉じた」を表す `weekBare:boolean` の最小構成。三値化せずboolを1つ足すのがポイント。

```
読書週間「桜の枝」を足して。
- 日〜土の7マスをグリッド表示。状態：done→🌸 / weekBareかつ未達→閉じた空きマス(点線) / それ以外→🪷
- マス押下で 未達⇄達成 をトグル。達成にした時だけ 和菓子+1・継続+1。hoverで微拡大、popアニメ
- 今日の曜日は枠を強調。下に「n/7日」進捗メッセージ
- 「今週の桜をリセット」ボタン：つぼみを全消し＝weekBare=trueの「閉じた状態」にする（つぼみへ戻さない）
- 週が変わったら weekDone を初期化し weekBare=false
状態は後述のstateに保存。
```

## フェーズ4：朝の一服／今日の読書／四段階手帖／和菓子棚

【解説】記録の中心。手帖は「定義→たとえ→詳細→一言まとめ」の四段階で、自分の言葉に落とすための型です。達成系はすべて和菓子・継続・称号に接続します。

```
記録セクションを足して。
- 朝の一服：1日1回ボタンで朝の一行を表示（十数パターン抽選、1日1回制御）
- 今日の読書：本タイトル・目標数・単位(ページ/分/章)。「読み終えた」で当日を達成扱い＋和菓子+1
- 学びの手帖：テーマ＋四段階(定義/たとえ/詳細/一言まとめ)を「綴じる」で保存。
  一覧表示、個別削除、チェックでの一括削除、空状態の案内文
- 和菓子棚：達成ごとに和菓子が増える木の棚。増えた要素にpop。称号テーブルで昇格時トースト。
  継続日数は「前回が昨日→+1、今日→据置、それ以外→1」
```

## フェーズ5：壊れない保存層（Store）

【解説】`localStorage` は容量超過・無効化で例外を投げます。失敗したらメモリに退避して継続し、ログに残す。これで「入力が一瞬で消える」事故を防ぎます。

```
保存層Storeを入れて。
- 起動時に書き込みテストして可否フラグを持つ
- get/set/remove/available。読み書きは必ずtry/catch
- set失敗時はメモリ(オブジェクト)へ退避し以降もメモリ動作、falseを返す。失敗はLoggerに記録
- removeあり。空キーは "null" を残さず完全削除
- state{wagashi,notes[],morningDate,readDate,weekStart,weekDone[7],weekBare,streak,lastDay} を
  load/saveで読み書き
```

## フェーズ6：書きかけ自動保存（InputDraft）

【解説】確定データ（手帖）とは別に、**入力中の生テキスト**を専用キーへ自動保存。`localStorage` なのでタブを閉じても残り、起動時に復元します。

```
書きかけ入力の自動保存InputDraftを入れて。
- 対象：本タイトル/目標/単位/テーマ/定義/たとえ/詳細/一言まとめ
- 確定データとは別キー。inputは600msデバウンス、change/beforeunload/visibilitychange(hidden)で即時保存
- 起動時にrestore、復元したらトースト通知＋「書きかけ自動保存中」ステータス表示
- 綴じる/読了/「入力を消す」で該当下書きをクリア。全項目空ならキーごと削除
```

## フェーズ7：消えないエラーログ（Logger）

【解説】不具合を画面と `localStorage` の両方に残し、サポート時に貼れるようにします。Logger は Store に依存させず自前で `localStorage` を触る（循環回避＋ストレージ障害でも動く）。

```
エラーログLoggerを入れて。
- window.error と unhandledrejection をグローバル捕捉
- 「時刻/文脈/メッセージ」で記録、最大50件、画面パネル＋localStorageに永続化
- 1件以上で自動表示、起動時に過去ログがあれば開く
- 表示/非表示・コピー・消去・疑似エラー(動作確認)ボタン
- 保存層に依存しない（自前でlocalStorageを直接触る）
```

## フェーズ8：一時保存／復元・Markdown出力

【解説】状態スナップショットの手動（メモ付き）／自動退避と、外部持ち出し用のMarkdown出力。バックアップ手段がローカルのみなので、MD出力が実質の“持ち出し口”になります。

```
ツールバーを足して。
- 一時保存：手動(メモ付き・時刻記録)と自動(30秒間隔＋visibilitychange＋beforeunload、デバウンス)
- 復元：手動/自動を選び、上書き前にconfirm。保存メタは除いて適用
- MD出力：称号・今週・手帖一覧を整形しBlob(text/markdown)でダウンロード（日付入りファイル名）
- 直近の保存状況をステータス表示
```

## フェーズ9：手帖の自動時間削除（TTL）

【解説】一時メモが無限に溜まらないよう、**作成からN日で自動消去**。各手帖に作成時刻を持たせ、起動時に「now − 作成 > TTL」を除外。破壊操作なのでONはconfirm＋遡及sweepにします。

```
手帖の自動消去を入れて。
- 各手帖に作成時刻(タイムスタンプ)を保持。TTLは定数(7日)で一箇所に
- 起動時に now-作成 > 7日 のものを除外（sweep）、件数をトースト通知。作成時刻が無い旧データは残す
- ON/OFFトグル：ON時にconfirm（7日経過分が外れる旨）→直後に既存の期限切れを即sweep。設定は保存
```

## フェーズ10：週次自動リセット（純粋関数＋取りこぼし救済）

【解説】「毎週日曜23:59:59」。ms算出・表示ラベル・タイマー標的が別計算だと将来ズレるので、**純粋関数 `nextSundayReset(now)` 1個に集約**。スリープ対策に、復帰時の取りこぼし救済も足します。

```
週次自動リセットを入れて。
- 純粋関数nextSundayReset(now)：次の日曜23:59:59のDateを返す。
  今日が日曜で時刻前なら当日、それ以外(同時刻ちょうど含む)は翌週。常に日曜を返す
- ms算出・画面の「次回自動リセット」ラベル・タイマー標的はすべてこの関数を参照
- 再帰setTimeoutで発火→桜を「閉じた状態」にリセット→再スケジュール
- 取りこぼし救済：visibilitychange(visible)とfocusで、標的を過ぎていたら実行して組み直す
- ローカル時刻ベース（JST想定）
```

## フェーズ11：カフェモード・花びら・トースト・レスポンシブ・a11y

【解説】仕上げの“空気感”と使い勝手。装飾は `aria-hidden`、通知領域は `aria-live` で最低限のアクセシビリティも入れます。

```
仕上げを足して。
- 和カフェモード：トグルでbodyにクラス付与し配色を上品に変化＋立ちのぼる湯気アニメ。状態を保存復元
- 背景に舞う桜の花びらを十数個、JSでサイズ/位置/速度をランダム生成（pointer-events:none・低z-index）
- トースト通知（画面下中央、数秒で消滅、連続呼び出しは上書き）
- 560px以下のレスポンシブ調整
- 純装飾SVG/演出はaria-hidden、意味ある画像はrole=img+aria-label、通知領域はaria-live=polite
```

## フェーズ12：商用テスト（jsdom）

【解説】DOM・localStorage・タイマーに結びつく挙動を、**実ファイルをjsdomに読み込んで操作模擬**で検証します。詳細手法はテスト記事参照。

```
このHTMLを実ファイルのままjsdomで読み込む商用テストを書いて。
- runScripts:"dangerously", pretendToBeVisual:true, url:"https://localhost/"、トップレベルはwindow経由で検証
- 入力はvalue設定＋input/changeディスパッチ、クリックは.click()
- 観点：書きかけ自動保存／別セッション復元(beforeParseでseed=タブ閉じ再現)／綴じ・読了・クリアで下書き消去／
  エラーログ表示・永続化／window.onerror捕捉／保存失敗時のメモリ退避(Storage.prototype.setItem差し替え)／
  桜リセット(つぼみ全消し＋DEBUG alertが出ない)／週次自動リセット(参照実装と総当たり＋境界＋取りこぼし救済)／既存機能の非破壊
- TZ=Asia/Tokyoで実行、末尾にPASS/FAIL件数。window.close()で後始末、FAILで非0終了
```

## 付録：理解のためのデータ地図

**localStorageキー**

| キー | 用途 |
|------|------|
| `usausa_techo_v1` | 確定状態(state本体) |
| `usausa_techo_input_v1` | 書きかけ入力 |
| `usausa_techo_errlog_v1` | エラーログ(最大50件) |
| `usausa_techo_draft_v1` / `usausa_techo_auto_v1` | 一時保存(手動/自動) |
| `usausa_autoerase_v1` | 手帖自動消去ON/OFF |

**state構造**

```
{ wagashi:number, notes:Note[], morningDate, readDate,
  weekStart, weekDone:boolean[7], weekBare:boolean, streak:number, lastDay }
Note = { id(作成時刻), title, def, meta, detail, sum, date }
```

【解説】このデータ地図を頭に入れておくと、どのプロンプトがどのキー/フィールドを触るかが分かり、改造時に迷いません。`weekBare` が「閉じた状態」、`id` が作成時刻＝TTL判定の基準、という2点が今回の肝です。
